Введение в Web программирование на PHP

Курс длится 1 семестр. В неделю 1 лекция она же практика. Спецкурс ориентирован на студентов КН, КБ, ФИИТ, ПИ второго курса, но приглашаются все желающие.
Проектор обеспечит факультет, а ноутбуки приносить свои. Желающие получат зачет или экзамен по результатам работы в семестре. Занятия проходят в 622 ауд. пятой парой (с 16-10 до 17-40), занятия начнутся 05 марта.

Цель преподавания дисциплины

Приобретение практических навыков разработки несложных Web приложений.
Знакомство с принципами разработки Web приложений.
Обзор некоторых библиотек языка PHP.
В рамках курса студенты имеют возможность взять курсовые работы, которые могут стать основой для бакалаврских и магистерских работ.

Задачи изучения дисциплины

Знакомство студентов с архитектурой Web сервера Apache.
Освоение студентами языка PHP в объёме, достаточном для решения практических задач.
Знакомство студентов с системой управления контентом Joomla!.

Место дисциплины в системе высшего профессионального образования

Данный спецкурс рассчитан на студентов вторго курса направления КН, КБ, ФИИТ, ПИ.
В рамках курса используются знания, полученные студентами на курсах "Скрипты", "Языки сценариев (Perl или Python)", "Сети". Курс является дополнением к курсам "Протоколы Интернета", "Web и DHTML", "Базы данных".
Базы данных изучаются на третьем курсе. В рамках спецкурса предполагается работа с базами данных. Основные понятия (необходимые для понимания решаемых задач), будут изложены на занятиях, а подробности будут оставлены желающим на самостоятельный разбор. Такой подход позволит познакомить студентов с базами данных уже на втором курсе.

Требования к уровню освоения содержания курса

Студент должен иметь представление о принципах организации сети Интернет и службы Web; о различных подходах к решению проблемы динамического контента; об архитектуре Model View Controller применительно к Web приложениям; о принципах работы CMS систем.

Студент должен знать синтаксис языка программирования PHP; архитектуру Web сервера Apache (структуру каталогов, директивы конфигурационных файлов, возможности ядра и модулей, интерфейсы и т. д.).

Студент должен уметь устанавливать, настраивать Web сервер Apache; писать PHP-скрипты для решения стандартных задач, использовать стандартные библиотеки; использовать возможности CMS Joomla! и её расширений. По окончании курса студент должен сделать свой сайт, движок которого написан на PHP.

Темы и разделы курса

  1. Основы языка HTML. Устройство Веб сервера.
  2. Обработка форм на стороне Веб сервера.
  3. Подключение к базам данных через веб интерфейс.
  4. Сеансы PHP и Cookie.
  5. Работа с библиотекой GD4. Captcha – защита от спам-роботов.
  6. Обработка XML документов.
  7. Системы управления контентом, Joomla.
  8. Отправка почты средствами PHP.
  9. Технология AJAX.
  10. Обзор библиотек и фреймворков для PHP. Архитектура MVC (Model View Controller).
В течение семестра можно сдать две задачи: написать свой форум и свою систему управления контентом; эти задачи в свою очередь состоят из маленьких подзадач: обработка форм, доступ к БД, регистрация и авторизация, защита от роботов (captcha, определение IP-адреса), поддержка сеансов, фильтр мата, рассылка писем и т. д.

Некоторые разобранные задачи

  1. Hello, world
  2. Обработка форм
  3. Созадние БД в PHPMyAdmin
  4. Создание БД с использованием интерфейса phpMyAdmin. Создали БД (имя БД дает сервер) и в ней таблицу students с четырьмя полями (колонками): id, name, address, birthday (идентификатор - INT, первичный ключ с автоинкрементированием; имя - VARCHAR$; адрес - VARCHAR; дата рождения - DATE). Внесли в таблицу данные.
  5. Подключение к БД, выборка всех содержащихся в таблице данных. Подключение вынесли в отдельный файл connection.php. Для вызова connection.php используем функцию require. Для выполнения SQL запроса испльзуем функцию mysql_query. Полученные данные вставляем на веб страницу в виде html-таблицы (тэг table). Исходники.
  6. Выборка данных, удовлетворяющих критерию поиска. В файле default.php создали веб форму (тэг form), атрибут action задает адрес и имя обработчика, атрибут method задает метод отправки данных на сервер (в данном примере метод GET). Текстовое поле для ввода (input type='text') имеет атрибут name со значением stud_name, следовательно, имя пользователя хранится в суперглобальном массиве $_GET. Для выборки строк, удовлетворяющих условию используется SQL конструкция where name='$_GET[stud_name]'. Поиск по возрасту делается аналогично. Исходники.
  7. Вставка данных в БД. Исходники.
  8. Сеансы PHP. В БД хранятся имена и md5-хэши паролей. Пользователь водит пароль и логин, а скрипт check.php проверят наличие в БД записи об этом пользователе. Если такая запись есть, заводится сессия - session_start(); и в суперглобальный массив $_SESSION делается запись о том, что пользователь успешно залогинился - $_SESSION['login']='ok'.
    В файле secret.php выполняется проверка. Если пользователь не залогинился, то выполняется редирект на страницу авторизации - header("Location:default.php"). Исходники
  9. Каптча. Пользователь получает форму для заполнения (разгадывания каптчи), рисунок формируется скриптом, что видно в атрибуте тэга img: src='captcha.php'. Основным скриптом, генерирующим рисунок, является captcha.php; он инклудит random.php для вынесения функции generate_code() в отдельный файл.
    Случайно сгенерированное с помощью generate_code() число заносится в сенанс: $_SESSION['captcha']=$captcha. Далее формируется рисуонк, который отправляется ползователю.
    Скрипт go.php запускает сессию, извлекает (если он там есть) из массива $_SESSION секретный код и сравнивает его с присланным пользователем.Исходники
  10. AJAX. Исходники.
  11. Почта. Исходники.
  12. XML. Исходники.
  13. Загрузка файлов на сервер. Исходники.
